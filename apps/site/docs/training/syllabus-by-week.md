# Syllabus by Week

This sample weekly plan is written for a fuller tutorial-style program. It starts before Flutter itself and ends with app quality, design clarity, and capstone delivery.

## Week 1: Programming basics

Focus:

- variables, input-output thinking, conditions, loops, and functions
- how to solve small logical problems step by step
- exercises that build confidence before app development starts

Expected outcome:

- learners stop seeing programming as abstract and start understanding simple logic flow

## Week 2: Dart fundamentals

Focus:

- Dart syntax, collections, null safety, functions, and classes
- object-oriented thinking for app development
- small practice problems using Dart only

Expected outcome:

- learners can write small useful Dart programs and understand the language behind Flutter better

## Week 3: Flutter foundations

Focus:

- widget tree, stateless and stateful widgets, and layout systems
- navigation and page structure
- building the first complete app flow

Expected outcome:

- learners begin building visible application flows instead of isolated experiments

## Week 4: UI building and app design

Focus:

- form creation, validation, and user interaction states
- layout discipline, spacing, hierarchy, and reusable components
- responsive UI thinking and better screen planning

Expected outcome:

- learners build cleaner, more usable interfaces with stronger structure

## Week 5: State and architecture

Focus:

- managing local and shared state
- organizing features and folders
- separating UI, logic, and data for cleaner apps

Expected outcome:

- learners begin to structure medium-sized apps more responsibly

## Week 6: APIs and data integration

Focus:

- making network requests and handling responses
- parsing JSON and creating app models
- authentication basics, persistence, and network-aware UI states

Expected outcome:

- learners understand how apps connect to real data and how to keep that flow organized

## Week 7: Quality and release readiness

Focus:

- debugging, testing basics, performance awareness, and build preparation
- cleaner coding habits and review-friendly structure
- release thinking for internal demos or production paths

Expected outcome:

- learners develop more professional habits around quality and delivery

## Week 8: Capstone and presentation

Focus:

- planning a complete mini app
- building, refining, and reviewing the implementation
- presenting the result with design and engineering rationale

Expected outcome:

- learners finish with a more meaningful project artifact and clearer explanation ability

## Optional extension weeks

- advanced state management and architecture
- richer animations, dashboards, and more complex flows
- live project mentoring and product-specific implementation support
