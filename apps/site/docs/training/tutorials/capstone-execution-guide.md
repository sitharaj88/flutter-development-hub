# Capstone Execution Guide

The capstone should feel like a small real product, not a random final assignment.

## Good capstone workflow

- choose a scope that can be completed well
- map the user flow before building
- define reusable UI pieces and data requirements
- build in checkpoints rather than waiting until the end
- review both design and engineering quality

## Presentation guidance

- explain the app problem clearly
- show the main user flow
- discuss design choices and technical tradeoffs
- describe what would be improved next with more time
