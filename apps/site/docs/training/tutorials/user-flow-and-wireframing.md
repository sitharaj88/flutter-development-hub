# User Flow and Wireframing

Strong apps are planned before they are coded. This tutorial introduces simple but practical design planning habits that help learners and teams move from idea to structure with less confusion.

## Lesson objective

By the end of this lesson, a learner should be able to:

- explain what a user flow is
- sketch simple screen relationships before development
- identify primary and secondary actions on a screen
- reduce clutter by planning earlier

## Topics

- mapping user goals
- sketching screen relationships
- identifying primary actions and supporting actions
- reducing clutter before development starts

## Plain-language explanation

A user flow is the path a person takes through the app to complete something meaningful.

Examples:

- sign in and reach the dashboard
- browse a course and enroll
- search, filter, and open a detail page

Wireframing helps turn that path into simple screen structure before full design or implementation begins.

## Why this matters

- implementation becomes faster when the flow is clear
- design discussion becomes more objective
- learners understand the relationship between UX planning and Flutter execution

## Good teaching workflow

1. define the user goal
2. list the screens or states involved
3. connect them in sequence
4. identify the most important action on each screen
5. sketch the layout roughly before coding

## What to emphasize in class

- wireframes are planning tools, not polished design work
- clarity matters more than visual detail at this stage
- the goal is to reduce uncertainty before implementation begins
