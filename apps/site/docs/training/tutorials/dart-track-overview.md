# Dart Track Overview

Dart is the programming language behind Flutter, but it should be taught as more than a prerequisite. A strong Dart track gives learners the confidence to model data, structure logic, handle async flows, and write code that stays understandable as the app grows.

## What this track covers

- variables, types, operators, and expressions
- functions, parameters, return values, and modular logic
- collections, classes, constructors, and object modeling
- null safety, generics, enums, and extension methods
- async programming, exceptions, and safe error handling
- JSON modeling, serialization thinking, and code organization

## Why this matters for Flutter

- widgets are easier when logic is easier
- app state is clearer when data is modeled well
- API integration depends on async and structured model classes
- maintainable teams need readable Dart before they need fancy Flutter patterns

## Recommended study order

1. variables, final, and const
2. operators and expressions
3. functions and parameters
4. null safety and types
5. collections and modeling
6. classes and constructors
7. inheritance, mixins, and interfaces
8. generics, enums, and extensions
9. async programming
10. exceptions, JSON, and code organization

## Phase-based teaching flow

- start with syntax and confidence-building
- reinforce logic through functions and small tasks
- introduce modeling before moving into app-heavy scenarios
- teach async and reliability once learners understand data flow
- finish with practice, review, and app-oriented Dart tasks

## What completion should look like

- learners can model app data cleanly
- learners can write readable utility and validation logic
- learners can reason about null safety, async flow, and errors
- learners are ready to use Dart confidently inside Flutter projects
