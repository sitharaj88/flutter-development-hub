# Dart Practice and Review

This page helps turn the Dart track into active learning instead of passive reading.

## Practice categories

- syntax and fundamentals
- functions and logic
- collections and modeling
- null safety and reliability
- async and error handling
- app-focused Dart exercises

## Good mini problems

- validate a sign-up form
- calculate totals from a cart list
- convert a JSON-like map into a clean model
- format user-facing labels and statuses
- simulate delayed data loading with `Future.delayed`

## Phase-based practice sequence

### Phase 1 and 2

- rewrite mutable declarations more clearly
- build validation helpers with named parameters
- create readable expressions for common app rules

### Phase 3 and 4

- choose correct collections for app scenarios
- model optional and required data safely
- write fallback logic for incomplete input

### Phase 5 and 6

- create model classes with constructors
- define a simple contract with an abstract class
- replace weak status strings with enums

### Phase 7 to 9

- simulate API loading with `Future`
- handle parsing errors safely
- convert JSON into a model and back again
- refactor a messy helper file into cleaner Dart organization

## Review questions

- when should you use `final` versus `const`?
- why do named parameters help readability?
- what problem does null safety solve?
- when should a `Map` become a class?
- when is a `Future` enough and when does a `Stream` make more sense?

## Suggested checkpoint style

- quick written review after each phase
- one mini lab at the end of every two phases
- one consolidation task that resembles app-ready Dart usage

## Consolidation goal

- learners should be able to solve medium-sized language problems without depending on UI code to guide them
