# Variables and Data Types

Variables are how programs remember things. This lesson helps learners understand what kinds of data exist and how to work with them safely.

## Lesson goal

- understand what a variable is
- identify common data types
- choose the right kind of data for a simple task
- understand why the wrong data shape creates confusion later

## What to teach

- text, numbers, booleans, lists, and maps
- mutable and fixed values
- naming variables clearly
- why wrong data types cause errors and confusion

## Plain-language explanation

You can explain a variable as:

- a named place where the program stores a value

Then explain data types as:

- the kind of value stored in that place

This helps learners connect names, meaning, and type more clearly.

## Real app examples

- username and email as text
- age, total, and rating as numbers
- isLoggedIn as a boolean
- product list as a collection

## Good classroom prompts

- should a cart total be text or number?
- why is login status better as true/false than as text?
- when should many values be stored as a list instead of separate variables?

## Practice ideas

- create variables for a sign-up form
- store product information using multiple data types
- explain why a number should not be stored as plain text unless necessary

## What learners should be able to do after this lesson

- name variables more clearly
- choose more sensible types for simple problems
- understand why data modeling matters in app development

## Teaching outcome

- learners gain confidence storing and reasoning about application data
