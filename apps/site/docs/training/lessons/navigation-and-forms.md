# Navigation and Forms

Most useful apps need screen flow and input. This lesson ties those ideas together.

## Lesson goal

- move between screens with clear structure
- gather and validate user input
- design basic app flows such as login, signup, or profile editing
- understand how forms and navigation shape the user journey

## What to teach

- screen transitions and route thinking
- passing information between screens
- text fields, validation, and form handling
- disabled, loading, success, and error behavior for forms

## Plain-language explanation

Navigation is how users move through tasks.

Forms are how users give information to the app.

Together they create many of the most important flows in real products.

## Practical build ideas

- login flow
- sign-up flow
- contact or feedback form
- list screen to detail screen navigation

## Good teaching sequence

1. define the user goal
2. map the screens involved
3. build the input form
4. validate and respond to user actions
5. navigate when the task succeeds

## What learners should be able to do after this lesson

- build a simple multi-screen flow
- validate a small form more clearly
- think about loading and error behavior, not just happy-path input

## Teaching outcome

- learners can build multi-screen apps with sensible form behavior and better UX
