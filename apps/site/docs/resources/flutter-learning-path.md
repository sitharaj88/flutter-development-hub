# Flutter Learning Path

This path is designed to help learners move in the **right order** instead of jumping between random topics. Each stage builds on the previous one.

:::tip How to use this
- **Students** — Follow sequentially as a learning map
- **Teams** — Identify the stage you need most and focus there
- **Institutions** — Use it to shape workshop or bootcamp depth
:::

## The seven stages

![Flutter Learning Path — 7 stages from Programming Foundations to Capstone](/img/diagrams/learning-path.svg)

---

### Stage 1: Programming foundations

| Topic | What you'll learn |
|-------|------------------|
| Problem solving | Breaking problems into steps |
| Control flow | `if`, `else`, loops, conditions |
| Functions | Parameters, return values, reuse |
| Data basics | Variables, types, collections |

**Goal:** Read and explain simple code confidently.

➡️ [Programming Fundamentals](/docs/training/programming-fundamentals)

---

### Stage 2: Dart in depth

| Topic | What you'll learn |
|-------|------------------|
| Types & null safety | Safe, expressive Dart code |
| Classes & constructors | Data modeling for apps |
| Collections | Lists, maps, sets, and transformations |
| Async programming | Futures, async/await, error handling |
| JSON & serialization | API data modeling |

**Goal:** Write Dart with enough confidence to focus on Flutter, not the language.

➡️ [Dart for Flutter](/docs/training/dart-for-flutter)

---

### Stage 3: Flutter UI foundations

| Topic | What you'll learn |
|-------|------------------|
| Widget tree | Composition and nesting |
| Layouts | Row, Column, Stack, constraints |
| Navigation | Push, pop, named routes |
| Forms | Input, validation, user flows |
| Theming | Reusable styles and components |

**Goal:** Turn a screen design into a working widget tree.

➡️ [Flutter Core](/docs/training/flutter-core)

---

### Stage 4: Design and UX

| Topic | What you'll learn |
|-------|------------------|
| Visual hierarchy | Spacing, typography, color |
| Interaction design | Touch targets, feedback, flow |
| Responsive design | Adapting to different screen sizes |
| Design systems | Consistent, reusable components |

**Goal:** Build UI that's usable, not just functional.

➡️ [App Design & UX](/docs/training/app-design-and-ux)

---

### Stage 5: Architecture and data

| Topic | What you'll learn |
|-------|------------------|
| State management | Local vs. shared state patterns |
| Project structure | Feature-based module organization |
| Data layer | APIs, repositories, persistence |
| Dependency injection | Clean, testable architecture |

**Goal:** Structure apps that scale without becoming unmaintainable.

➡️ [State & Architecture](/docs/training/state-and-architecture)

---

### Stage 6: Testing and delivery

| Topic | What you'll learn |
|-------|------------------|
| Unit testing | Logic verification |
| Widget testing | UI behavior testing |
| Debugging | Issue isolation and diagnosis |
| Release prep | Build, sign, and deploy |

**Goal:** Ship with confidence, not crossed fingers.

➡️ [Testing & Release](/docs/training/testing-and-release)

---

### Stage 7: Capstone and proof

| Topic | What you'll learn |
|-------|------------------|
| Guided project | End-to-end app development |
| Code review | Quality assessment and feedback |
| Portfolio prep | Showcase-ready documentation |
| Case study | Presenting your work professionally |

**Goal:** Prove your skills with a real, reviewable project.

➡️ [Capstone](/docs/training/capstone)
