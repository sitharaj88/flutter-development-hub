Flutter engineer, trainer, and technical consultant with deep experience in mobile and cross-platform application development.

## What I do

| Area | Focus |

| **Flutter Training** | Structured programs from foundations to production delivery for students, teams, and institutions |
| **Technical Consulting** | Architecture guidance, code reviews, delivery acceleration, and technical rescue for Flutter projects |

## Engineering focus areas

- **Cross-platform Flutter** — iOS, Android, web (WASM), and desktop from a single codebase
- **Architecture design** — Feature-based structure, Riverpod state management, repository pattern, clean separation
- **Modern Dart 3** — Sealed classes, records, pattern matching, and code generation (`freezed`, `riverpod_generator`)
- **Quality engineering** — Testing strategy, CI/CD pipelines, performance optimization, and release automation
- **Team capability building** — Turning individual developers into confident, independent Flutter teams

## Working approach

- Explain complex concepts clearly and calmly — no jargon walls
- Connect training directly to real implementation, not abstract theory
- Leave behind reusable artifacts — standards documents, starter repos, architecture guides
- Focus on making teams **self-sufficient**, not dependent on external help
